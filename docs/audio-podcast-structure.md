# 三、每个模块的详细指南

## 模块1：开场Hook

**目的**：前5秒决定观众走不走

**已有资源**：

- 12种开场版本（A-L）
- 开场Hook生成器提示词

**时长**：1-2分钟

**关键点**：

- 第一句话必须抓人
- 不要说"大家好"、"欢迎来到"这种废话
- 直接进入冲击性内容

---

## 模块2：本期预告

**目的**：告诉观众会得到什么，值得听下去

**模板**：

```
今天讲[人物/事件]。
讲[X]个真相：
第一，[真相1核心]
第二，[真相2核心]
第三，[真相3核心]
听完你会明白[核心价值]。
```

**示例**：

```
今天讲褚时健。
讲5个真相：
第一，他是被设计好的牺牲品
第二，能干为什么是一种罪
第三，这不是个案，是系统
第四，出路在哪
第五，你的脑子润出来了吗
听完你会明白：问题不在你，在系统。
```

---

## 模块3：背景铺垫

**目的**：给新观众必要信息

**已有资源**：

- 6种铺垫风格
- 背景铺垫生成器提示词

**关键点**：

- 50岁观众不需要太多铺垫
- 边讲边带比单独铺垫好
- 可以用"你可能知道，但你不知道这个"结构

---

## 模块4-8：核心真相

**目的**：脚本核心，你的价值所在

**结构**：

```
【真相抛出】
一句话说清楚这个真相是什么

【证据支撑】
- 数据
- 案例
- 对比
- 引用

【Q问DY答】
穿插Q的提问，让节奏更活

【真相强化】
用金句收尾
```

**关键点**：

- 每个真相要有"别人不说"的感觉
- 要有证据，不能空口白话
- 短句为主，每句不超过15字
- 每个真相结尾要有金句

---

## 模块5-9：拆解

**目的**：分析"为什么"，这是丁元英风格的核心

**结构**：

```
【提问】
你问为什么？

【系统分析】
因为[系统层面的原因]

【规律总结】
这个规律是[XX]

【丁元英引用】（可选）
丁元英怎么说来着？"[语录]"

【回到案例】
[人物]就是这样
```

**关键点**：

- 不止说what，要说why
- 要上升到系统层面，不是就事论事
- 要揭示规律，不是描述现象

---

## 模块10：意义延伸

**目的**：告诉观众"跟你有什么关系"

**结构**：

```
【转向观众】
现在聊聊屏幕前的你

【身份定位】
如果你在海外...
如果你在体制里待过...

【三层关系】
1. 对你的资产意味着什么
2. 对你的家庭意味着什么
3. 对你的思维意味着什么

【自检问题】
问你几个问题，不用回答我：
1. [问题1]
2. [问题2]
3. [问题3]
```

---

## 模块11：收尾

**目的**：总结+行动+订阅

**结构**：

```
【一句话总结】
今天的核心是什么？就一句话：[核心观点]

【行动指引】
如果这期对你有启发，你可以：
1. [行动1]
2. [行动2]

【点赞订阅】
如果觉得有价值，点个赞。
让更多人看到。
我们下期见。
```

| 模块 | 建议语速 | 原因 |

| 位置 | 停顿时长 |
|------|----------|
| 金句前 | 1秒 |
| 金句后 | 2秒 |

| 内容 | 音量 |
|------|------|
| 正常内容 | 中等 |
| 重点强调 | 略高 |

# 五、Q角色使用指南

## Q的作用

| 作用 | 说明 |
|------|------|
| 打断节奏 | 避免单口播客太闷 |
| 代表观众 | 问出观众心里的问题 |
| 推进内容 | 引出下一个话题 |
| 增加可听性 | 两个声音比一个好听 |

## Q的话术风格

| 类型 | 示例 |
|------|------|
| 困惑 | "等等，这是什么意思？" |
| 追问 | "那他为什么要这样做？" |
| 质疑 | "你是说他被人搞了？" |
| 总结 | "所以你的意思是..." |
| 连接 | "那这跟我们有什么关系？" |

## Q的分布

| 时间段 | Q出现频率 |
|--------|----------|
| 前15分钟 | 每3-5分钟1次 |
| 中间 | 每5-8分钟1次 |
| 后15分钟 | 减少，让DY收尾 |

# 八、转场技巧

## 8.1 模块间转场类型

| 转场类型 | 适用场景 | 示例 |
|----------|----------|------|
| **总结式** | 真相→拆解 | "这是事实。问题是，为什么会这样？" |
| **递进式** | 真相1→真相2 | "这只是第一层。还有更狠的。" |
| **问题式** | 拆解→下一真相 | "搞清楚了这个，还有一个问题..." |
| **对比式** | 铺垫→真相 | "这是官方说法。真相呢？" |
| **挂钩式** | 任意模块 | "前面讲的，都是在铺垫。接下来才是重点。" |

## 8.2 转场话术模板库

### 开场Hook → 本期预告

```
这个故事，今天我们来拆。
讲[X]个真相。
```

### 本期预告 → 背景铺垫

```
在进入正题之前，先聊几个基本信息。
（或直接跳过，边讲边带背景）
```

### 背景铺垫 → 真相1

```
这些你可能知道。
但你不知道的是——
```

### 真相1 → 拆解1

```
这是事实。
问题是，为什么会这样？
```

### 拆解1 → 真相2

```
这是第一个真相。
还有更狠的。
（或：但这只是开始。）
```

### 真相 → 意义延伸

```
讲完了[人物]的故事。
现在聊聊屏幕前的你。
```

### 意义延伸 → 收尾

```
问题问完了。
最后，总结一下今天的核心。
```

## 8.3 转场禁忌

| 禁忌 | 原因 | 替代 |
|------|------|------|
| "接下来我们讲..." | 太刻板 | "还有一个问题..." |
| "下面进入第二部分" | 像PPT汇报 | "这只是第一层" |
| 无过渡直接跳 | 观众跟不上 | 至少用一句话连接 |
| 过渡太长 | 节奏拖沓 | 一两句话足够 |

| 15-25分钟 | 💡高点1 | 拆解1高潮，金句落地 |

| 30-50分钟 | 📈再上升 | 真相2+3，持续深入 |

## 9.3 情绪控制技巧

| 要升情绪 | 方法 |
|----------|------|

| 用反问 | "你觉得公平吗？" |
| 加速语速 | 节奏快=情绪紧 |

| 要降情绪 | 方法 |
|----------|------|
| Q提问打断 | "等等，这是什么意思？" |
| 放慢语速 | 给观众消化时间 |
| 讲小细节 | 从宏观到微观 |
| 开小玩笑 | 调节气氛 |

# 十、完整示例脚本（真相模块）

## 10.1 真相模块完整示例

以下是一个完整的"核心真相+拆解"示例：

```
【核心真相1：他是被设计好的牺牲品】

DY：

褚时健出事，官方说是贪腐。

1000多万。

听起来不少，对吧？

但你想想——

他给国家赚了多少？

16年，累计上千亿。

他拿了多少？

1000多万。

差不到万分之一。

Q：

那他为什么还要拿？

DY：

因为他67岁了。

快退休了。

退休之后呢？

什么都没有。

16年，把一个破厂干成中国第一。

然后拍拍屁股走人。

一分钱的股权没有。

一分钱的分红没有。

企业的增值，跟他没有一毛钱关系。

**这就是国企的设计。**

你创造价值。

系统拿走价值。

你是工具，不是所有者。

任正非为什么能成首富？

因为他有股权。

柳传志为什么能功成身退？

因为他是联想的主人。

褚时健呢？

他什么都不是。

他只是一个"高级打工仔"。

干得再好，都是给别人做嫁衣。

**这是第一个真相：**

**在某些系统里，你越能干，你被剥削得越狠。**

---

【拆解1：系统为什么要这样设计？】

DY：

你问为什么？

因为任何系统，首先考虑的不是效率，是稳定。

稳定靠什么？

靠控制。

控制靠什么？

靠资源分配。

如果能人可以积累财富，他就有了独立的资本。

有了资本，就有了话语权。

有了话语权，就可能挑战系统。

所以系统的逻辑是：

**你可以有能力，但你不能有资本。**

**你可以创造价值，但价值必须归系统。**

这不是阴谋，这是结构。

丁元英怎么说的？

"这是文化属性决定的，不以任何人的意志为转移。"

褚时健不是输给了某个人。

他是输给了一套系统逻辑。

---

【转场 → 真相2】

DY：

这是第一个真相：制度设计让他必然被牺牲。

但还有更狠的。

你以为能干就能出头？

错了。

在某些地方，能干是危险的。

这是第二个真相。
```

# 十一、不同系列的结构差异

## 11.1 资本往事 vs 历史镜像 vs 元英说理

| 维度 | 资本往事 | 历史镜像 | 元英说理 |
|------|----------|----------|----------|
| **时长** | 60-120分钟 | 45-90分钟 | 30-60分钟 |
| **真相数量** | 3-5个 | 2-3个 | 1-2个核心观点 |
| **铺垫比例** | 10% | 15-20% | 5% |
| **故事比例** | 50% | 60% | 20% |
| **分析比例** | 30% | 25% | 60% |
| **意义延伸** | 必须有 | 必须有 | 融入正文 |
| **Q角色** | 频繁 | 中等 | 较少 |
| **情绪曲线** | 大起大落 | 渐进上升 | 平稳深入 |

## 11.2 资本往事专属结构

```
适用于：人物传记类（褚时健、马云、任正非）

```

# 十二、听众疲劳点处理

## 12.1 关键疲劳点

| 时间点 | 疲劳原因 | 处理方法 |
|--------|----------|----------|
| **15分钟** | 新鲜感消退 | 抛出第一个金句/高潮 |
| **30分钟** | 第一波疲劳 | 互动点/Q提问打断 |
| **45分钟** | 信息过载 | 小总结，让观众喘口气 |
| **60分钟** | 明显疲劳 | 第二高潮或换话题 |
| **90分钟** | 深度疲劳 | 开始收束，别再加新内容 |

## 12.2 疲劳点处理话术

### 15分钟处理

```
DY：好，基础讲完了。
接下来才是重点。
（制造期待感）
```

### 30分钟处理

```
DY：如果你听到这里觉得有收获，
点个赞，让更多人看到。
我们继续。
（短暂喘息+互动）
```

### 45分钟处理

```
Q：等等，先别往下讲。
我先消化一下刚才说的。
所以你的意思是...
（让Q做总结，帮观众整理）
```

### 60分钟处理

```
DY：前面讲了这么多，
都是在铺垫一件事。
接下来这个，是今天最重要的。
（重新抓注意力）
```

# 十三、内容密度控制

## 13.1 什么是内容密度？

内容密度 = 单位时间内的信息量

**密度太高**：观众跟不上，听完什么都没记住
**密度太低**：观众觉得啰嗦，直接划走

## 13.2 理想密度配比

| 内容类型 | 建议比例 |
|----------|----------|
| 核心观点 | 20% |
| 证据/案例 | 30% |
| 分析/拆解 | 25% |
| 重复/强调 | 15% |
| 过渡/喘息 | 10% |

## 13.3 一期内容放多少真相？

| 时长 | 建议真相数 | 原因 |
|------|-----------|------|
| 30分钟 | 1-2个 | 讲透比讲多重要 |
| 60分钟 | 2-3个 | 留足拆解时间 |
| 90分钟 | 3-4个 | 可以深入展开 |
| 120分钟 | 4-5个 | 每个真相15-20分钟 |

## 13.4 密度控制技巧

| 要增密度 | 方法 |
|----------|------|
| 加数据 | 用具体数字增加信息量 |
| 加案例 | 用故事承载观点 |
| 加对比 | 对比增加理解深度 |

| 要降密度 | 方法 |
|----------|------|
| 重复金句 | 核心观点说2-3遍 |
| 加Q提问 | 打断节奏，帮观众消化 |
| 加总结 | 每个模块结尾小总结 |
| 加停顿 | 给观众思考时间 |

# 十四、失败案例分析

## 14.1 结构问题导致的失败

### 失败案例1：没有开场Hook

```
❌ 错误开场：
"大家好，欢迎来到元英视角，我是DY，今天我们来聊一聊褚时健这个人物..."

问题：10秒内就会被划走

✅ 正确开场：
"200亿。一个人，一年，给国家交200亿的税。拿多少？20万。"
```

### 失败案例2：铺垫太长

```
❌ 错误：前15分钟都在讲背景

问题：观众等不到重点就走了

✅ 正确：铺垫控制在3-5分钟，或边讲边带
```

### 失败案例3：只讲what不讲why

```
❌ 错误：讲完褚时健的故事就结束了

问题：没有价值感，听完觉得"然后呢"

✅ 正确：每个真相后面要有拆解，解释为什么
```

### 失败案例4：没有意义延伸

```
❌ 错误：讲完别人的故事就收尾

问题：观众觉得"跟我有什么关系"

✅ 正确：必须有"这跟你有什么关系"的模块
```

### 失败案例5：收尾太sudden

```
❌ 错误："好，今天就到这里，拜拜"

问题：结束太突然，没有回味

✅ 正确：一句话总结 + 行动指引 + 点赞订阅
```

## 14.2 自检清单

录制前用这个清单检查脚本：

- [ ] 开场前5秒有没有冲击力？
- [ ] 铺垫是不是太长了？
- [ ] 每个真相有没有拆解"为什么"？
- [ ] 有没有意义延伸模块？
- [ ] 收尾有没有一句话总结？
- [ ] 金句分布均匀吗？
- [ ] Q的出现频率合理吗？
- [ ] 有没有处理疲劳点？
- [ ] 转场是不是自然？
- [ ] 内容密度合适吗？

# 十五、开场后的过渡技巧

## 15.1 开场Hook → 正文的六种过渡

### 过渡1：直接预告（最常用）

```
Hook: "200亿 vs 20万，10万倍的差距。"
过渡: "今天讲5个真相。第一..."
```

### 过渡2：设问过渡

```
Hook: "71岁进监狱，84岁成橙王。"
过渡: "他是怎么做到的？答案可能跟你想的不一样。"
```

### 过渡3：否定常见认知

```
Hook: "很多人讲他励志。"
过渡: "我不讲。今天讲真相。"
```

### 过渡4：悬念延续

```
Hook: "他为什么会出事？"
过渡: "要回答这个问题，先得知道一件事。"
```

### 过渡5：时间回溯

```
Hook: "1999年，他被判无期。"
过渡: "让我们把时间倒回1979年..."
```

### 过渡6：观众定位

```
Hook: "这个故事，没人敢在国内讲。"
过渡: "如果你在海外，你需要知道这个。"
```
